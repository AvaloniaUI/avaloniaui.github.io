Remarks

The XAML engine applies its bindings in a delayed manner where bindings are only applied when a control is added to the visual tree. This was done because applying bindings as soon as controls are created means that long-form bindings (i.e. bindings that don't use the {Binding} markup extension) don't work as the binding is applied to the property before the binding properties are set, and looking at WPF it uses a similar mechanism for bindings that come from XAML.
